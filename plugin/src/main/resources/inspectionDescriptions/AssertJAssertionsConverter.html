Reports calls to Assert.assertEquals(), Assert.assertTrue(), etc. methods which can be migrated to AssertJ's self-describing Assertions.assertThat() chain calls.

For example:


    public class SubstantialTest {
        @Test
        public void test() {
            assertTrue("Foobar" instanceof String);
            assertFalse(new HashMap().size() > 1);
            assertEquals("foo", "foobar".substring(0, 3));
            assertNull(null, "Null must be null");
            assertNotNull(new Object());
        }
    }

A quick-fix is provided to perform the migration:


    public class SubstantialTest {
        @Test
        public void testContents(Collection<String> c, String s) {
            assertThat("Foobar").isInstanceOf(String.class);
            assertThat(new HashMap()).hasSizeLessThanOrEqualTo(1);
            assertThat("foobar".substring(0, 3)).isEqualTo("foo");
            assertThat(null).as("Null must be null").isNull();
            assertThat(new Object()).isNotNull();
        }
    }

This inspection requires that the AssertJ library is available on the classpath.

Use the Statically import AssertJ's methods option to specify if you want the quick-fix to statically import the AssertJ methods.
